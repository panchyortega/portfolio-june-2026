# Proyectos futuros

Banco de ideas de **proyectos de diseño** que Panchy querría hacer en
algún momento — chicos o grandes, concretos o todavía vagos. El objetivo
es tener un lugar de referencia para cuando quiera ponerse a diseñar algo
nuevo pero no sepa qué: revisar esta carpeta antes de partir de cero.

## En qué se diferencia de `ideas/`

Este repo tiene **dos** carpetas de ideas, con propósitos distintos:

| Carpeta | Es sobre... | Ejemplo |
|---|---|---|
| [`ideas/`](../ideas/) | Features y arquitectura **de este sitio** (el portafolio en sí) | "agregar diagramas Mermaid", "reorganizar la home en categorías" |
| `proyectos-futuros/` (esta carpeta) | **Proyectos de diseño** que Panchy quiere hacer — posibles casos de estudio futuros, no tienen nada que ver con el código del sitio | "rediseñar tal app", "explorar tal patrón de interacción" |

Si la idea es sobre cómo funciona o se ve *este sitio web*, va en `ideas/`.
Si la idea es un proyecto de diseño que Panchy podría trabajar y
eventualmente subir como caso de estudio, va acá.

## Por qué existe

Documentar bien una idea de proyecto — el problema que resuelve, qué la
hace interesante, referencias, alcance aproximado — toma criterio y
tiempo. Si esa reflexión no queda escrita, se pierde, y la próxima vez
hay que pensarla de nuevo desde cero (o peor, se olvida que existió).

## Cómo se usa

**Para agregar una idea nueva:**
1. Crear un archivo `.md` nuevo en esta carpeta, nombre corto y
   descriptivo en kebab-case (ej: `rediseno-app-clima.md`).
2. Documentar lo que haya disponible — no hace falta tenerlo todo
   pensado, la idea puede quedar bien vaga. Como guía, conviene incluir
   lo que se tenga de esto:
   - **Qué es / de dónde salió** — la inspiración, el problema que
     llamó la atención, o simplemente la ocurrencia.
   - **Por qué interesa** — qué la hace atractiva de trabajar (un reto
     de UX puntual, ganas de explorar cierto estilo visual, practicar
     una herramienta, etc.).
   - **Alcance aproximado** — ¿es un ejercicio rápido de un fin de
     semana, o un proyecto grande tipo case study completo?
   - **Referencias** — links, capturas, ejemplos que la inspiraron.
   - **Estado** — algo simple como "solo la idea", "boceteado en
     Figma", "en pausa", etc.
3. No hay que preocuparse por dejarla "bien armada" — a diferencia de
   `ideas/` (que suele documentar una conversación ya explorada a
   fondo), acá puede quedar una idea de una línea. Lo importante es que
   quede registrada.

**Para retomar una idea:**
Revisar esta carpeta como punto de partida cuando haya ganas de diseñar
algo pero no haya claridad de qué. No hace falta elegir la idea "más
desarrollada" — cualquiera puede ser el punto de partida.

**Cuando una idea se convierte en proyecto real:**
No se borra el archivo. Se le agrega una línea de estado
(`**Estado:** convertido en proyecto — ver /proyectos/slug-del-proyecto`)
para dejar el registro de dónde nació.
